# Level 7 — ModuleMesh（Boss 关卡）

> 最终关卡：方法级别 Attribute 扫描、两个生成器协同、跨程序集模块通信注册表模式。

---

## 你会学到

- **`MethodDeclarationSyntax`**：如何在谓词中命中方法级别的 Attribute，而非类或字段
- **两个生成器协同工作**：`SchemaGenerator` 提取通信协议信息，`ModuleCodeGenerator` 读取协议生成胶水代码，两者通过约定的数据结构解耦
- **跨程序集模块通信模式**：调用方不依赖被调用方的具体类型，只通过生成的 `ModuleMeshRegistry` 进行事件派发（Emit）和工具调用（Call），实现真正的模块解耦

---

## 核心概念

### 两阶段生成：Schema → Glue Code

ModuleMesh 的核心思想是把"描述通信接口"和"生成通信胶水"分成两个独立的生成步骤：

**阶段一 — SchemaGenerator（提取协议）**

扫描所有带 `[Event]` 或 `[Tool]` 的方法，提取方法名、参数类型、所在模块名，输出一个描述文件（或通过 `AdditionalTextsProvider` / `IncrementalValueProvider` 传递数据）。

```
GameModule.OnPlayerDied([Event])  →  EventSchema { Name="OnPlayerDied", Module="GameModule" }
GameModule.GetScore([Tool])       →  ToolSchema   { Name="GetScore",     Module="GameModule" }
```

**阶段二 — ModuleCodeGenerator（生成胶水代码）**

读取阶段一的 Schema，生成 `ModuleMeshRegistry` 静态类，包含：

- `Register(module)` — 模块自注册，把自己的事件处理器和工具实现绑定到注册表
- `Emit(eventId, data)` — 向所有已注册的模块广播事件
- `Call<TResult>(toolId, args)` — 向特定模块调用工具方法并返回结果

**为什么要解耦？**

假设 `UIModule` 想知道玩家死亡：

```
❌ 传统耦合方式：UIModule 直接引用 GameModule，调用 gameModule.OnPlayerDied(data)
✓  ModuleMesh 方式：UIModule 调用 Registry.Emit(GameEvents.OnPlayerDied, data)
```

`UIModule` 的代码完全不知道 `GameModule` 的存在。只要 `GameModule` 已注册，它就能收到事件。模块可以随时插拔，不影响其他模块。

### 方法级 Attribute 的语法树位置

```
MethodDeclarationSyntax  void OnPlayerDied(PlayerData data)
  ├─ AttributeListSyntax
  │    └─ AttributeSyntax [Event("OnPlayerDied")]
  │         └─ ArgumentListSyntax
  │              └─ AttributeArgumentSyntax "OnPlayerDied"
  ├─ ReturnType: void
  ├─ Identifier: OnPlayerDied
  └─ ParameterList: (PlayerData data)
```

从 `AttributeSyntax` 的 `ArgumentList` 中可以提取事件/工具的名称和其他元数据。

## 关键代码

### 谓词：命中带 Attribute 的方法

```csharp
context.SyntaxProvider.CreateSyntaxProvider(
    predicate: static (node, _) =>
        node is MethodDeclarationSyntax m
        && m.AttributeLists.Count > 0,   // ★ 有 Attribute 的方法
    transform: static (ctx, _) => ExtractMethodSchema(ctx)
)
```

### 从方法 Attribute 提取 EventSchema / ToolSchema

```csharp
private static MethodSchema? ExtractMethodSchema(GeneratorSyntaxContext ctx)
{
    var method = (MethodDeclarationSyntax)ctx.Node;

    foreach (var attrList in method.AttributeLists)
    foreach (var attr in attrList.Attributes)
    {
        var attrName = attr.Name.ToString();

        if (attrName is "Event" or "EventAttribute")
        {
            // 提取事件名（第一个字符串参数，或默认使用方法名）
            var eventName = GetFirstStringArg(attr) ?? method.Identifier.Text;

            // 提取包含类（模块）名
            var moduleName = (method.Parent as ClassDeclarationSyntax)?.Identifier.Text ?? "Unknown";

            return new MethodSchema(SchemaKind.Event, eventName, moduleName,
                ExtractParameters(method, ctx.SemanticModel));
        }

        if (attrName is "Tool" or "ToolAttribute")
        {
            var toolName = GetFirstStringArg(attr) ?? method.Identifier.Text;
            var moduleName = (method.Parent as ClassDeclarationSyntax)?.Identifier.Text ?? "Unknown";

            return new MethodSchema(SchemaKind.Tool, toolName, moduleName,
                ExtractParameters(method, ctx.SemanticModel));
        }
    }

    return null;
}

private static string? GetFirstStringArg(AttributeSyntax attr)
{
    var firstArg = attr.ArgumentList?.Arguments.FirstOrDefault();
    if (firstArg?.Expression is LiteralExpressionSyntax lit
        && lit.IsKind(SyntaxKind.StringLiteralExpression))
        return lit.Token.ValueText;
    return null;
}
```

### 生成 `ModuleMeshRegistry`

```csharp
private static void GenerateRegistry(
    SourceProductionContext ctx,
    ImmutableArray<MethodSchema?> schemas)
{
    var events = schemas.Where(s => s?.Kind == SchemaKind.Event).ToList();
    var tools  = schemas.Where(s => s?.Kind == SchemaKind.Tool).ToList();

    var sb = new StringBuilder();
    sb.AppendLine("// <auto-generated/>");
    sb.AppendLine("public static partial class ModuleMeshRegistry");
    sb.AppendLine("{");

    // 为每个 Event 生成分发方法
    foreach (var ev in events)
    {
        if (ev is null) continue;
        var paramList = string.Join(", ", ev.Parameters.Select(p => $"{p.TypeName} {p.Name}"));
        sb.AppendLine($"    // Event: {ev.Name} (来自 {ev.ModuleName})");
        sb.AppendLine($"    public static void Emit_{ev.Name}({paramList})");
        sb.AppendLine($"    {{");
        sb.AppendLine($"        foreach (var module in _registeredModules)");
        sb.AppendLine($"            (module as I{ev.Name}Handler)?.On{ev.Name}({string.Join(", ", ev.Parameters.Select(p => p.Name))});");
        sb.AppendLine($"    }}");
        sb.AppendLine();
    }

    // 为每个 Tool 生成调用代理
    foreach (var tool in tools)
    {
        if (tool is null) continue;
        sb.AppendLine($"    // Tool: {tool.Name} (来自 {tool.ModuleName})");
        sb.AppendLine($"    public static object? Call_{tool.Name}(params object[] args)");
        sb.AppendLine($"    {{");
        sb.AppendLine($"        var provider = _registeredModules.OfType<I{tool.Name}Provider>().FirstOrDefault();");
        sb.AppendLine($"        return provider?.{tool.Name}(args);");
        sb.AppendLine($"    }}");
        sb.AppendLine();
    }

    sb.AppendLine("}");
    ctx.AddSource("ModuleMeshRegistry.g.cs", sb.ToString());
}
```

## 在 Unity 中运行

1. 打开 `Assets/Showcase/Level7/Level7Demo.cs`，查看两个模块的定义：

   ```csharp
   // GameModule.cs
   public partial class GameModule : IModule
   {
       [Event] public void OnPlayerDied(PlayerData data) { ... }
       [Tool]  public int  GetScore() => _score;
   }

   // UIModule.cs
   public partial class UIModule : IModule
   {
       void Start()
       {
           // UIModule 完全不引用 GameModule
           ModuleMeshRegistry.Emit_OnPlayerDied(new PlayerData { Name = "Hero" });
           int score = (int)ModuleMeshRegistry.Call_GetScore()!;
           Debug.Log($"Final Score: {score}");
       }
   }
   ```

2. 展开 Analyzers，查看 `ModuleMeshRegistry.g.cs`，确认生成了 `Emit_OnPlayerDied` 和 `Call_GetScore` 方法。

3. 运行场景，Console 输出：

   ```
   [GameModule] Player died: Hero
   [UIModule] Final Score: 42
   ```

4. 尝试添加一个新的 `[Event]` 方法，保存后观察 `ModuleMeshRegistry.g.cs` 自动更新。
