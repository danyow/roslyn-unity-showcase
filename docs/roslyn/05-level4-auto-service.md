# Level 4 — AutoService Generator with Scriban

> 第四关：用 `.Collect()` 把多个类聚合为一批，再用 Scriban 模板引擎渲染汇总代码，告别 `StringBuilder` 拼字符串的时代。

---

## 你会学到

- **Scriban 模板引擎**：用 `{{~ for item in services ~}}` 语法在 `.scriban` 文件中描述代码结构，让生成逻辑与模板分离
- **`.Collect()` 聚合**：将多个独立的 `IncrementalValueProvider<T>` 合并为 `IncrementalValueProvider<ImmutableArray<T>>`，实现"扫描 N 个类，只触发一次生成"
- **EmbeddedResource 模式**：把 `.scriban` 模板嵌入到生成器程序集，运行时通过 `Assembly.GetManifestResourceStream()` 读取，无需文件路径依赖

---

## 核心概念

### 逐类生成 vs 聚合生成

前三关的模式是**逐类生成**：每发现一个符合条件的类，就生成一个对应的源文件。

```
[AutoService] AudioService  →  AudioService.g.cs
[AutoService] InputService  →  InputService.g.cs
[AutoService] NetworkService→  NetworkService.g.cs
```

但 AutoService 的目标是生成**一个**统一的 `Service` 入口类：

```csharp
public static class Service
{
    public static AudioService AudioService { get; } = new AudioService();
    public static InputService InputService { get; } = new InputService();
    public static NetworkService NetworkService { get; } = new NetworkService();
}
```

这需要**先收集所有服务类，再一次性生成**。`.Collect()` 正是为此而生。

### `.Collect()` 的工作方式

```csharp
// provider 是 IncrementalValuesProvider<ServiceInfo>（多值）
var provider = context.SyntaxProvider.CreateSyntaxProvider(...);

// .Collect() 转为单值：IncrementalValueProvider<ImmutableArray<ServiceInfo>>
var collected = provider.Collect();

// 现在注册生成，回调只会被调用一次，收到全部服务信息
context.RegisterSourceOutput(collected, GenerateServiceClass);
```

增量编译的优化也保留：只有当 `ImmutableArray<ServiceInfo>` 整体内容发生变化时，生成步骤才会重新执行。

### Scriban 模板语法简介

Scriban 使用 `{{~ ... ~}}` 标签（两侧的 `~` 用于吞掉多余空白行）：

```scriban
// 自动生成，请勿手动修改
public static class Service
{
{{~ for svc in services ~}}
    public static {{ svc.type_name }} {{ svc.property_name }} { get; }
        = new {{ svc.type_name }}();
{{~ end ~}}
}
```

与 `StringBuilder` 相比，模板让"代码的形状"一目了然，维护起来更轻松。

### 为什么 Scriban.dll 也需要 Analyzer 标签？

Roslyn 分析器/生成器运行在编译器的进程里，**不是**最终应用程序的一部分。如果 `Scriban.dll` 只是普通的 `<PackageReference>`，它会被复制到项目输出目录，但编译器进程找不到它，运行时抛 `FileNotFoundException`。

正确配置：

```xml
<PackageReference Include="Scriban" Version="5.x.x">
    <PrivateAssets>all</PrivateAssets>
    <IncludeAssets>runtime; build; native; contentfiles; analyzers</IncludeAssets>
</PackageReference>
```

或在 Generator 项目里：

```xml
<ProjectReference Include="..\MyGenerator\MyGenerator.csproj">
    <OutputItemType>Analyzer</OutputItemType>
    <ReferenceOutputAssembly>false</ReferenceOutputAssembly>
</ProjectReference>
```

`ReferenceOutputAssembly>false` 确保 Scriban.dll 不会被传递到消费项目，只供生成器自用。

## 关键代码

### `.Collect()` 聚合与生成

```csharp
public void Initialize(IncrementalGeneratorInitializationContext context)
{
    context.RegisterPostInitializationOutput(ctx =>
        ctx.AddSource("AutoServiceAttribute.g.cs", AutoServiceAttributeSource));

    var services = context.SyntaxProvider.CreateSyntaxProvider(
        predicate: static (node, _) => node is ClassDeclarationSyntax cls
                                       && cls.AttributeLists.Count > 0,
        transform: static (ctx, _) => GetServiceInfo(ctx)
    ).Where(x => x is not null);

    // ★ 关键：聚合所有服务
    var collected = services.Collect();

    context.RegisterSourceOutput(collected, GenerateServiceClass);
}
```

### 读取 EmbeddedResource 模板

```csharp
private static string LoadTemplate()
{
    var assembly = typeof(AutoServiceGenerator).Assembly;
    using var stream = assembly.GetManifestResourceStream(
        "MyGenerator.Templates.ServiceTemplate.scriban")!;
    using var reader = new StreamReader(stream);
    return reader.ReadToEnd();
}
```

### Scriban 渲染

```csharp
private static void GenerateServiceClass(
    SourceProductionContext ctx,
    ImmutableArray<ServiceInfo?> services)
{
    var validServices = services.Where(s => s is not null).ToList();
    if (validServices.Count == 0) return;

    var templateText = LoadTemplate();
    var template = Template.Parse(templateText);

    var scriptObj = new ScriptObject();
    scriptObj["services"] = validServices;

    var templateCtx = new TemplateContext();
    templateCtx.PushGlobal(scriptObj);

    var output = template.Render(templateCtx);
    ctx.AddSource("Service.g.cs", output);
}
```

### `.scriban` 模板文件（`ServiceTemplate.scriban`）

```scriban
// <auto-generated/>
// 此文件由 AutoServiceGenerator 自动生成，请勿手动修改。
public static partial class Service
{
{{~ for svc in services ~}}
    /// <summary>{{ svc.display_name }} 服务单例</summary>
    public static {{ svc.type_name }} {{ svc.property_name }} { get; }
        = new {{ svc.type_name }}();

{{~ end ~}}
}
```

## 在 Unity 中运行

1. 打开 `Assets/Showcase/Level4/Level4Demo.cs`，场景启动时会调用：

   ```csharp
   Service.AudioService.Play("bgm");
   bool jumping = Service.InputService.IsJumping();
   ```

2. 在 Rider 中展开 Analyzers，查看 `Service.g.cs`，确认所有 `[AutoService]` 类都被聚合为 `Service` 的静态属性。

3. 运行场景，Console 输出：

   ```
   [AudioService] Playing: bgm
   [InputService] IsJumping: False
   ```
